2. Try Babel

Babel is a free and open-source JavaScript transcompiler that is mainly used to convert ECMAScript 2015+ (ES6+) code into a backwards compatible version of JavaScript that can be run by older JavaScript engines.

How to use babel in civetjs framework

1. Move your uncompiled javascript file to 'uncompile' folder on root

2. type npm run build on cmd to compile your scripts and move the scripts to 'dist' folder on root

3. see the result with npm run serve which is automatically directed to 'dist/index.js'

NOT RECOMMENDED - HIGH MEMORY USAGE : You can also do npm run start which is automatically directed to 'index.js' on root folder

You can change the destination file and destination folder in package.json

3. Try Webpack

Webpack is a static module bundler for modern JavaScript applications. When webpack processes your application, it internally builds a dependency graph which maps every module your project needs and generates one or more bundles.

How to use webpack in civetjs framework

1. Copy your file / entry with the filename index(default) that need to be bundled to 'src/'

2. type npm run webpack on cmd to bundle the file and move the file to 'dist/' with the filename bundle(default)

You can change default bundle filename, entry filename and destination in webpack.config.js
